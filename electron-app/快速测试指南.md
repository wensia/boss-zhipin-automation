# 快速测试指南

## 🎯 目标

测试 Electron 桌面版本，**不影响现有程序的正常运行**。

## ✅ 测试步骤

### 方法 1：使用快速启动脚本（推荐）

```bash
# 1. 确保后端和前端正在运行
# 终端 1
cd backend
uv run python -m app.main

# 终端 2
cd frontend
npm run dev

# 2. 使用快速启动脚本测试 Electron
cd electron-app
./start-dev.sh
```

### 方法 2：手动启动

```bash
# 1. 安装 Electron 依赖（只需首次执行）
cd electron-app
npm install

# 2. 确保后端运行在 http://localhost:27421
# 3. 确保前端运行在 http://localhost:13601

# 4. 启动 Electron
npm start
```

## 🔍 预期结果

- 会打开一个新的桌面应用窗口
- 窗口中显示你的 Boss直聘自动化界面
- 功能与浏览器访问完全一致

## ⚠️ 重要说明

### 开发模式特点

- ✅ **不会打包**：只是在窗口中显示现有的网页
- ✅ **不影响现有服务**：使用的是已运行的后端和前端
- ✅ **可以同时运行**：浏览器和桌面版可以同时使用
- ✅ **支持热重载**：修改代码后自动刷新

### 测试建议

1. **先用浏览器确认功能正常**
   - 访问 http://localhost:13601
   - 确保登录、任务等功能正常

2. **再启动 Electron 测试**
   - 使用 `./start-dev.sh` 启动
   - 检查界面是否正常显示
   - 测试各项功能

3. **关闭 Electron**
   - 直接关闭窗口即可
   - 不影响后端和前端继续运行


```bash

```

**解决：**
```bash
cd electron-app

```

### 问题 4：窗口显示空白

**检查：**
1. 前端服务是否正常运行？
2. 访问 http://localhost:13601 是否正常？
3. 检查 Electron 控制台是否有错误

## 💡 提示

- 开发模式下，所有修改都会实时反映到 Electron 窗口中
- 可以打开多个 Electron 窗口进行测试
- 按 `Cmd+Q` 退出 Electron 应用
- Electron 窗口可以调整大小、最小化等，就像普通 Mac 应用

## 📞 需要帮助？

如果遇到问题：
1. 查看终端日志输出
2. 检查后端日志：`backend/` 目录
3. 检查前端控制台：浏览器开发者工具
4. 检查 Electron 日志：启动时的终端输出
